CROSS-CHANNEL DUPLICATE CHECK — SETUP
=====================================

WHAT THIS DOES
--------------
Rule o7 says the same advert may not be posted in more than one channel. This
command watches the advert channels and, when someone posts, compares the new
post against that user's current ads in the OTHER advert channels. If it finds
one that is the same (ignoring case, spacing, punctuation, and links), it pings
the poster in #rule_infractions asking them to differentiate or remove one.

This is a "Tier 1" check: it catches verbatim / copy-paste reposts (the most
common case staff handle by hand). It does NOT try to judge "reworded but
basically the same" posts — that is left to staff.


HOW IT FINDS THE USER'S ADS
---------------------------
It does NOT need any new data. Every advert command already stores
lastMsg_<channelID> = the message ID of the user's current ad in that channel,
keyed under the user's ID. One database query (dbGetPattern) returns all of
them, and the command re-reads each ad live to compare. If the user deleted an
ad, that read simply returns nothing and is skipped.

Because it pings through the shared "Alert Sender" command, a post that the
advert command already deleted (for length, cooldown, or a same-channel
duplicate) will NOT produce a duplicate ping.


BEFORE YOU START
----------------
1. Set up the shared "Alert Sender" command first (see alert_sender/setup.txt)
   and have its command ID ready. #rule_infractions is configured there, not
   here.


CREATE THE "Cross-Channel Duplicate Check" COMMAND
--------------------------------------------------
1. YAGPDB dashboard → Custom Commands → New Command.
2. Trigger type: Regex
3. Trigger: ([\s\S]*)
4. Channel Restrictions: Check off 'Only run in the following channels'
   then add ALL of your advert channels — quick, 1x1, and group.
5. Open cross_channel_dupes.go and copy its contents into the command's
   code box.
6. In that pasted code, set:
       {{ $alertSender := 0 }}
   By replacing the 0 with the "Alert Sender" command ID.
7. Save the command.
