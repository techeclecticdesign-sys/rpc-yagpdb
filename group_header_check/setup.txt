GROUP HEADER CHECK — SETUP
==========================

WHAT THIS DOES
--------------
In a group advert, ONE header line is allowed, and its text has a length cap:
    #   ->  50 characters max
    ##  ->  60 characters max
    ### ->  70 characters max

If a post in a group advert channel has more than one header line, OR a header
that is longer than its cap, the bot pings the poster in #rule_infractions and
asks them to fix it. A single header within its cap is allowed and triggers
nothing.


BEFORE YOU START
----------------
1. Turn on Discord Developer Mode:
   User Settings → Developer → Developer Mode (ON).
   This lets you right-click a channel and "Copy Channel ID".

2. Copy this channel ID (right-click channel → Copy Channel ID):
   • #rule_infractions  (where the reminder is posted)


CREATE THE "Group Header Check" COMMAND
---------------------------------------
1. YAGPDB dashboard → Custom Commands → New Command.
2. Trigger type: Regex
3. Trigger: (^|\n)#{1,3}[ ]
   (Fires when any line of the post is a header — i.e. starts with 1 to 3
    # marks followed by a space, whether it's the first line or further
    down. [ ] is a space; using a character class keeps it from being lost
    as a trailing space when pasting.)
4. Channel Restrictions: Check off 'Only run in the following channels'
   then add your group advert channels to the channel list below.
5. Open group_header_check.go and copy its contents into the command's
   code box.
6. In that pasted code, set:
       {{ $infractionsChannel := 0 }}
   By replacing the 0 with a paste of #rule_infractions channel ID.
7. Save the command.
