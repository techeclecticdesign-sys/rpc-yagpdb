QUICK REACTIONS — SETUP
=======================

WHAT THIS DOES
--------------
When someone posts in #quick_fandoms or #quick_originals, a timer
starts for that post. When the timer ends, the bot checks the post for at
least 3 unique reactions from the approved roleplay-reaction list. If the post
has fewer than 3, the bot posts a reminder in #rule_infractions that pings the
author.

This is built from TWO YAGPDB custom commands, each in its own file:
  • "Post Timer"     (post_timer.go)    — starts the timer on every new post
  • "Reaction Check" (reaction_check.go) — runs when the timer ends and does
                                           the check

You will create "Reaction Check" first (so you have its ID), then "Post Timer",
then fill in a couple of IDs.


BEFORE YOU START
----------------
1. Turn on Discord Developer Mode:
   User Settings → Developer → Developer Mode (ON).
   This lets you right-click a channel and "Copy Channel ID".

2. Copy these channel IDs (right-click channel → Copy Channel ID):
   • #rule_infractions  (where reminders are posted)
   • #advert_rules      (linked in the reminder text)


STEP 1 — CREATE THE "Reaction Check" COMMAND
--------------------------------------------
1. YAGPDB dashboard → Custom Commands → New Command.
2. Trigger type: None
   (This should NEVER trigger on its own, only when "Post Timer" calls it.)
3. Open reaction_check.go and copy its contents into the command's code box.
4. In that pasted code, set:
       {{ $infractionsChannel := 0 }}
   By replacing the 0 with a paste of #rule_infractions channel ID.
5. Also in the pasted code, set:
       {{ $advertRules := 0 }}
   Replace the 0 with a paste of #advert_rules channel ID.
5b. Also set:
       {{ $infractionsSticky := 0 }}
   Replace the 0 with the "#rule_infractions sticky" command ID (see the
   infractions_sticky setup). This lets the sticky drop back below each
   reminder this command posts. If you leave it 0, the reminder still sends —
   the sticky just won't re-stick beneath it.
6. Save the command.
6. Find this command's ID: open/edit it in YAGPDB dashboard and look at the
   page URL — it ends in a number.
   e.g. .../customcommands/commands/1234/  → the ID is 1234.
   Write this number down; you need it in Step 2.


STEP 2 — CREATE THE "Post Timer" COMMAND
----------------------------------------
1. YAGPDB dashboard → Custom Commands → New Command.
2. Trigger type: Regex
3. Trigger: (?s).*
   (This matches every new message in the allowed channels.)
4. Channel Restrictions: Check off 'Only run in the following channels'
   then add #quick_fandoms and #quick_originals to the channel list below.
5. Open post_timer.go and copy its contents into the command's code box.
6. In that pasted code, set:
       {{ $checkCC := 0 }}
   Replace the 0 with the "Reaction Check" command ID you wrote down in Step 1.
   And, if you want a different wait time, change:
       {{ $delaySeconds := 300 }}
   300 seconds = 5 minutes. Change this number to adjust how long the bot
   waits before checking a post.
7. Save the command.
